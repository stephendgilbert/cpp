The cat Filter

To learn about text filters, first click the "running man" icon to log into your online version of VSCode.

Once you're logged in, type the following commands in the shell to upload the starter code for this lesson. (Type the portion that appears after prompt $, and make sure you press ENTER afer typing each line.)

$ wget https://occ-cs.com/filters.zip
$ unzip filters.zip
$ cd filters
filters/ $ code input.txt moby.txt filter.cpp

Now, let's look at the most common built-in filter command. Type the following command in the shell (terminal), and press ENTER. (Again, type only the portion that appears after the $ prompt.)

filters/ $ cat

The cursor simply blinks; you don't get a new prompt. Go ahead and type a few lines of text, pressing ENTER at the end of each line. The input you typed is echoed on the next line. Press CTRL+D to return to the prompt.

  • Filter programs read from standard input and write to standard output.
  • The cat filter concatenates each input character to standard output. In Windows, the equivalent filter is named type.
  • The filter stops reading when it reaches end-of-file. In Unix, you simulate that by typing CTRL+D from the terminal. In Windows, it is CTRL+Z.

A filter is not meant to be run interactively. Instead, it is meant to process a stream of data that is supplied from a file, a network stream or some other source. The easiest way to supply such a stream is to use input redirection.
