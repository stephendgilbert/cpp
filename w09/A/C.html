What Should You Throw?

Unlike Java, in C++, it is legal to throw any kind of object, not just members of an exception class hierarchy. So, in C++, all of these are legal: 

if (len < 3) throw "Too short"s; // throw a string
if (a > b) throw 42;    // throw an integer error code
if (b < c) throw 3.5;   // throw a double

The question is, though, what should stoi() throw when an error occurs? The library documentation says that the function throws two different kinds of exceptions:

  • An invalid_argument exception when the argument cannot be converted to an int.
  • An out_of_range exception when the converted argument would be too big to fit into an int.
The header file, <stdexcept> defines both of these (and several other classes) that let us specify what specific error triggered the exception, similar to the Exception class hierarchy from the Java Class Libraries. (Click the image to enlarge it.)

Both of these exceptions:

  • have a constructor which takes a string argument, useful for error messages.
  • have a member function what() that returns what the error was

Include <stdexcept>, and rewrite the throw statements like this: 

if (in.eof()) 
    throw out_of_range("str + " is too large for an int.");
else
    throw invalid_argument(str + " not an int.");

The first check here, if (in.eof())... is thrown when all of the string has been converted to integers, but the resulting integer is to large to fit into an int. The second is thrown when the string cannot be converted at all.
